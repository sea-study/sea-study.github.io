【Hexo】快速搭建博客

2025/11/29

使用【Hexo】快速搭建博客

  • 说明(简述)：
  • 我们部署主要需要git, github , hexo , node.js，cloudflare

一、所需工具的 - 检查 / 安装

先检查一下所需到底有没有安装过，再进行安装

如下图所示:

命令:
node -v
npm -v
git -v

查看是否已经安装过了，如果没有安装过，就看下面，如果已经安装直接检查git配置就行

git 安装地址

https://git-scm.com/

node.js 安装地址

https://nodejs.org/

注意：安装完之后为了保险起见，我们仍旧应该检查一遍是否安装成功，重启终端之后再输入 检查版本的命令

如果有版本号了，就证明成功了

配置 Git

git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"
设置完可以输入git config --global --list检查是否配置成功

说明：这里我的user.name和我的github是不一样的，这就相当于寄快递，只要是我的注册邮箱(地址)是正确的就行，不影响部署，但建议保持一致

安装Hexo

点一下 “安装Hexo”就可以进入Hexo的官网了

点开官网就可以看到许多命令了，安装命令是直接可以在终端里面输 npm install -g hexo-cli （这个命令一进官网就有）

  • 这里扩展一下，npm 是node的一个包管理器，下完node.js 自己会有npm命令，想深究可以自己去搜一下

二、创建博客项目

建个文件夹存放我们所需项目

这里我演示的文件夹叫做 Hexo_show_teach

我们可以直接在目录那里输入cmd进入终端

然后回到此文件夹的上一级目录 cd ..
再进行初始化命令
hexo init Hexo_show_teach
Hexo_show_teach(替换你自己想初始化的文件夹的名称)

说明：上面在终端的命令当然也可以在vscode里面的终端运行，两者运行都一样，后面的很多命令建议打开vscode,在里面的终端进行运行

![]

配置完成之后我们会发现我们原来空荡荡的文件夹现在变得满满当当

文件说明：

Hexo_show_teach
├── _config.yml # 网站的 主配置文件，非常重> 要！
├── source # 存放你的文章和页面
│ └── _posts # 你的博客文章都放在这里
├── themes # 存放主题的地方
├── scaffolds # 模板文件夹
├── public # 存放生成的静态文件(我们现在还没生成)
└── package.json # 应用程序的信息和依赖

看完上面的这个文件分布图，我们呢Hexo的这个基本框架就算创建好了

三、本地预览

1.通过命令hexo s（hexo server的简写）即可进行预览

如果出现如图下这种错误，大概率是默认的配置文件的ESM 与 CJS 模块系统不兼容，如果没有错误我们就可以进行写博客的操作了

这时候我们打开文件package.json我们可以看到我们应用程序的信息和依赖，问题就出在strip-ansi的版本不兼容。

我们打开这个文件之后，全删掉，将下面我修改过后的粘贴进去就行
{
“name”: “hexo-site”,
“version”: “0.0.0”,
“private”: true,
“scripts”: {
“build”: “hexo generate”,
“clean”: “hexo clean”,
“deploy”: “hexo deploy”,
“server”: “hexo server”
},
“hexo”: {
“version”: “7.3.0”
},
“dependencies”: {
“hexo”: “^7.0.0”,
“hexo-deployer-git”: “^4.0.0”,
“hexo-generator-archive”: “^2.0.0”,
“hexo-generator-category”: “^2.0.0”,
“hexo-generator-index”: “^3.0.0”,
“hexo-generator-tag”: “^2.0.0”,
“hexo-renderer-ejs”: “^2.0.0”,
“hexo-renderer-marked”: “^6.0.0”,
“hexo-renderer-stylus”: “^3.0.0”,
“hexo-server”: “^3.0.0”,
“hexo-theme-landscape”: “^1.0.0”
}
}

粘贴完之后 输入命令：
rm -r -fo node_modules（删除旧依赖）
rm -fo package-lock.json
npm i(安装依赖)

这里我写的都是简写的命令

然后再运行 hexo s就可以本地运行了

ctrl + c中止

2.我们的第一个博客

我们可以通过直接在路径 source/_posts 下新建一个.md文件，也可以在终端中输入命令hexo new 后面接上我们的项目名

* 设置主题

我们可以去官网找我们喜欢的主题并设置

选择一个我们喜欢的点进去

进入这个界面之后复制一下这个网页的 url 就行


当然，也可以直接看这个下面的readme，一般都会有说明

复制完的网址 通过使用 git git clone 命令下载到themes目录下的主题名的文件夹下(这个目录下面)
所以完整命令应该是
git clone https://github.com/raytaylorlin/hexo-theme-raytaylorism.git themes/raytaylorism

然后不能忘记在_config.xml里面修改主题噢

修改完成之后,保存一下 hexo clean hexo g hexo s就可以查看了，这三个命令我们以后写，会经常用到

hexo clean是清理旧文件(会有很多莫名其妙的问题)
通过这个命令清理旧文件就会少很多问题，也可以更新
hexo g是将我们的.md文件生成为静态网站
hexo s就是本地预览了

下面这就是配置完主题之后的样子了，刚才我选的比较简洁了

四、部署到 GitHub Pages

这是最后一步，让我们的博客可以被全世界看到

新建库

打开我们的github,登录之后点上面的新建

新建进去之后，注意命名规范和选择Public

安装部署插件

在博客根目录，安装 hexo-deployer-git
npm install hexo-deployer-git --save

部署完成之后再打开我们的_config.xml主文件，到最下面找到deploy进行下图这样的配置

配置完成之后
使用依次三条命令
hexo clean
hexo generate
hexo deploy

输入完成之后就如下图一样了，我们创的github库里面就有我们的文件了

这样我们就已经马上完成了，我们就可以直接访问
我们用户名+github.io

我们先不访问，先回来建了一个这个新的
然后依旧输入**hexo clean**
hexo generate
hexo deploy
这三条命令

这样就ok了，最后这张是我在我自己的博客上放的，因为对于域名的话GitHub是只给每个用户一个免费的域名的

就是我们用户名+github.io，我没买其他的域名，所以只用这个我免费的域名给大家展示了

总结

  • Node.js是运行JavaScript的环境,如果没有它，就无法安装和运行Hexo

  • Git 是让我们的博客可以部署到GitHub Pages， 还有，Hexo主题都是通过Git从GitHub上下载的

  • Hexo 那就不用说了，我这篇文章就是用Markdown格式写的，最后展现给你的是网页的形式，所以它可以通过.md文件快速生成静态网站

注：

在操作的过程当中，我们会有很多简单的
windows 系统 win + v 可以看我们的剪切板

MS office 免费安装配置

2025/11/23

MS office 免费安装配置 _ （保姆级教学！！）

前言

我们当初买电脑的时候可能都会出现一个问题，买电脑的官方会给我们一个限时’’免费’’的MS office全家桶，有的一年，有的两年，等到两年之后，全家桶失效了，系统会自动给我们装一个WPS的office，甚至是不装，就是突然用不了什么word,excel，ppt了，然后我们就傻眼了，因为都用了很长时间的MS了，突然一换，我们会很不熟悉，而再想弄成MS，就需要买了。所以我写这篇博客就是想让看到这篇文章的人能够学会白嫖MS office 全家桶。

一.安装前注意

如果电脑已经是重装系统的话可以直接跳到第二步

点此直接跳转到步骤二

为了**避免冲突**，我们的电脑必须是没有装过office的，先卸载之前的office,WPS也是，也要卸载，即使它不是微软下的ms

卸载office的方法我举了两种方式:

1.通过Geek卸载

我们可以直接在edge浏览器上面下一个，因为下载这个特别好用的工具，我们平常使用电脑也会用到

链接在此： https://geekuninstaller.com/

下载之后直接 在里面搜索office就行

索引到直接点击卸载就行

2.通过系统自带的卸载工具卸载

在终端输入 appwiz.cpl 命令即可调出系统自带的卸载工具了

也可以直接 win+i 打开设置然后点开应用去卸载也是可以的

二. 部署工作

1.准备工作（check it）

检查我们的office是否卸载完全

> 直接返回桌面，右键新建，如果还是有诸如
> - word,excel,ppt 的可新建的文件类型的话，
>多半是没有成功卸载，或者卸载不干净，请重复上述卸载步骤

检查完毕卸载完全进行以下步骤

1.在微软官网上下载部署工具

链 接: https://www.microsoft.com/en-us/download/details.aspx?id=49117

2.在桌面上创建一个文件夹，命名为office，使用部署工具下载文件，选择新建的文件夹将部署文件下载到该文件夹中
3.使用微软office自定义工具

https://config.office.com/deploymentsettings(链接)

设置相关配置，选择自己想配置的类型（最好选有批量许可证的），其他选择语音包，前面全默认，导出文件名改成config，将其复制到office文件夹里面

在配置的这个步骤也需要注意： 如果没那么多的需求，可以只勾选word,excel,ppt，因为如果勾选过多后面需要等待下载很长时间，下载时间越长，越容易出现问题

注意:

查看自己电脑是多少位的可以在终端（cmd）里面输入systeminfo在系统配置里面可以查看x64就是64位，如果是x32就是32位，大多数电脑是64位，就是x64

4.将文件夹的路径复制使用管理员身份进入终端（cmd）使用cd进行访问，然后
  • 下载命令:
    setup /download config.xml
    • 安装命令:
      setup /configure config.xml

依次输入，下载并安装office然后等待最后一步激活

注意

这步骤的时候需要注意！！ 下载和安装的时候可能会出现卡顿，下载的时候如果超过10分钟仍在加载，可能会出现以下两种 原因:

  • 1. 网络问题，没联网，ctrl+c中止任务，然后联网重新下载安装就可以。
  • 2. 卡顿问题，有的时候就是下载时间需要很长很长，这时候我们可以直接中止任务，但是下载时间不能太短，如果太短，是可以安装的，但是可能会出现虽然已经安装了，某个文件类型在使用的过程中会不被电脑所识别

注意！

—–扩展命令ctl在windows系统中是清除缓存的，如果需要重新下载，可以先清除缓存，再进行我们的下载安装任务

做完这步的时候我们就可以使用新建一个office的任意文件，比如说word，然后点击查看账户，应该是未激活状态，

三、最后一步

！（使用管理员权限在终端进行操作）！

    1. 安装目录内找到 ospp.vbs 文件路径
      cd C:\Program Files\Microsoft Office\Office16 (64 位)
      cd C:\Program Files (x86)\Microsoft Office\Office16(32 位)
    1. cscript ospp.vbs /sethst:kms.03k.org
    1. cscript ospp.vbs /act 激活

最后，和安装完之后的步骤一样，随便新建一个office文件，(就是word,excel,ppt之类的)，然后点击账户，如果是以下这样就证明激活成功啦！！ 恭喜您拥有了白嫖的永久的office ！！！

附言：

除非您重新装机，否则office不会莫名其妙过期消失

！！！最重要的噢，配置完成之后，可以把桌面上的配置文件删掉了哦，下次用的话再次看教程重新下就行咯，不会影响正常使用

目前教程还暂时没有配太多图文，因为作者是早早白嫖的，白嫖到的是24年最新版本的，现在白嫖都是旧版本的了，所以作者的图片没有演示全过程，图文描述很多都是文，能白嫖到就好，图片说明下次再进行更新。
